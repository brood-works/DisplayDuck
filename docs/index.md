# DisplayDuck docs

This docs site is the starting point for building and maintaining DisplayDuck packs.

If you are creating a new widget pack, start here:

- [Create a DisplayDuck pack](packs/getting-started.md)
- [Local development](packs/local-development.md)
- [Pack runtime reference](packs/runtime-reference.md)

## What these docs cover

- how to start a new pack from the example pack
- what `package.json` and `widget.json` values need to be set
- what each manifest field is for
- how `dist/` is produced
- what template helpers and placeholders are available
- what SCSS variables are currently safe to rely on
- how local development and watch mode work

## Terminology

DisplayDuck uses the term **pack**.

## Suggested reading order

1. Read [Create a DisplayDuck pack](packs/getting-started.md)
2. Set up [Local development](packs/local-development.md)
3. Use [Pack runtime reference](packs/runtime-reference.md) while building

## Docs status

This is an initial set of docs, intended to give pack authors a solid starting point. It is okay for this to grow over time as more runtime APIs, SCSS variables, and authoring conventions get formalized.
